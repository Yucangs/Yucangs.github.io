Minimun Size Subarray Sum

不想解释…

my solution

简单思考了一下, 以每个元素为起点, 循环往右找到满足条件的数

最终做比较, 就可以得出答案

这样的复杂度大概是 O(n2), 同时为了区别特殊情况, 还做了一次…

the best solution

其实这是我将算法思想重写后的代码…

简单来说, 就是先找到匹配的 sum, 然后没加一次, 就尝试减少前面的数

这样遍历之后, 得到的数同样也是最短的

(我很好奇没有是否没有人对数级的解决方案…)
